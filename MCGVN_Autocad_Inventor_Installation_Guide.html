Inventor Add-in Installation

01

Store Core Files

Place each plugin inside its own MCG_* subfolder, alongside the bat file:

C:\CustomTools\Inventor\MCG_*\

The bat will scan each subfolder and copy any *.addin   *.dll   *.idw files inside — file names are flexible.

02

Execute Batch File

Locate and run the following script to install the add-ins:

Install_AutoLoadInventorAddin.bat

📌 What does the bat do?

  1. Scans every MCG_* folder in the same directory as the bat file.
  2. Copies all .addin + .dll + .idw files inside each folder.
  3. Pastes them into %APPDATA%\Autodesk\Inventor 2023\Addins\
⚠️ Updating an add-in? The bat overwrites existing files in the target folder. You must close Inventor before running, otherwise the loaded .dll files will be locked and cannot be replaced.
📄 Install_AutoLoadInventorAddin.bat

File Copy Rules & Folder Structure


    
    

      
📝 The 3 Core Copy Rules

      
The bat scans every MCG_* subfolder and applies these rules to decide what to copy:

      

        

          
            

              File Type
              Required?
              Name Pattern
              Behavior
            

          
          
            

              *.addin
              YES — required
              Any name (no prefix needed)
              All .addin files in folder are copied. If none exist → folder is skipped.
            

            

              *.dll
              YES — required
              Any name (no prefix needed)
              All .dll files in folder are copied. If none exist → folder is skipped.
            

            

              *.idw
              NO — optional
              Any name (no prefix needed)
              All .idw files copied if found. If none → continues silently.
            

          
        

      

      

        💡 Key insight: File names inside each folder do NOT need to match the folder name. The bat scans by extension (*.addin, *.dll, *.idw), so a folder named MCG_InventorPickDataPlugin\ can contain MCG_InventorPickData.addin and it works fine.
      

    


    
    

      
📁 Recommended Folder Structure

      
Place the bat at the root, with each plugin in its own MCG_* subfolder:

      

        
C:\CustomTools\Inventor\

        
├── Install_AutoLoadInventorAddin.bat  ← run this

        
│

        
├── MCG_InventorSymbolHandler\

        
├── MCG_InventorSymbolHandler.addin

        
├── MCG_InventorSymbolHandler.dll

        
└── SymbolTemplate.idw  ← any name, optional

        
│

        
├── MCG_InventorPickDataPlugin\  ← folder uses long name

        
├── MCG_InventorPickData.addin  ← file uses short name → still works!

        
└── MCG_InventorPickData.dll

        
│

        
└── MCG_InventorCreateDummyDetailSection\

        
├── MCG_InventorCreateDummyDetailSection.addin

        
├── MCG_InventorCreateDummyDetailSection.dll

        
└── Symbol Sections.idw

      

      

        ✅ Validation: A folder is treated as a valid plugin only if it contains at least 1 .addin file AND at least 1 .dll file. Otherwise the bat skips it with a clear warning.
      

    


    
    

      

        ⚠️ Don't run from Google Drive / OneDrive
      

      
Running the bat from a cloud-synced virtual drive (Google Drive for Desktop, OneDrive, Dropbox) often fails silently. Three reasons:

      

        

          1. Files are "online only" — Cloud drives keep files as cloud placeholders. copy may fail or copy 0-byte files because the actual content isn't on disk yet.
        

        

          2. for /D may skip folders — Virtual filesystems don't enumerate folders the same way as NTFS. Some MCG_* folders might be silently skipped.
        

        

          3. Path resolution issues — %~dp0 on virtual drives can return unusual paths (e.g. G:\My Drive\... with spaces) that break script logic.
        

      

      

        ✅ Solution: Always copy the entire folder to a local drive first, then run the bat from there:
        

          xcopy "G:\My Drive\Custom Tools\Inventor" "C:\CustomTools\Inventor" /E /I /Y

          cd /d C:\CustomTools\Inventor

          Install_AutoLoadInventorAddin.bat
        

      

    


    
    

      

        
📝 Naming & Structure Rules

        

          

            RULE 1
            

              
Folder must start with MCG_

              
The bat uses MCG_* wildcard to discover plugin folders. Folders without this prefix are ignored.

            

          

          

            RULE 2
            

              
File names are flexible

              
.addin, .dll, .idw files inside the folder can have any name. They don't need to match the folder name or have MCG_ prefix.

            

          

          

            RULE 3
            

              
<Assembly> tag must point to an existing .dll

              
Inside the .addin XML, the <Assembly> value must match a .dll filename in the same folder. Mismatch = Inventor won't load the plugin.

            

          

          

            RULE 4
            

              
Unique GUID per plugin

              
Each .addin needs a unique ClassId/ClientId. Generate via PowerShell: [guid]::NewGuid()

            

          

          

            RULE 5
            

              
No spaces or special chars in folder/file names

              
Use underscores. Avoid !, ^, &, (, ) — they break delayed expansion in the bat.

            

          

        

      


      

        
✅ Valid vs ❌ Invalid Examples

        

          

            
✅ VALID — same name everywhere

            

              MCG_SymbolHandler\

                MCG_SymbolHandler.addin

                MCG_SymbolHandler.dll
            

          

          

            
✅ VALID — folder ≠ file name (still works)

            

              MCG_InventorPickDataPlugin\

                MCG_InventorPickData.addin

                MCG_InventorPickData.dll
            

          

          

            
✅ VALID — with optional .idw (any name)

            

              MCG_DetailSection\

                MCG_CreateDummyDetail.addin

                MCG_CreateDummyDetail.dll

                Symbol Sections.idw
            

          

          

            
❌ INVALID — folder doesn't start with MCG_

            

              SymbolHandler\  ← skipped (no MCG_ prefix)

                MCG_SymbolHandler.addin

                MCG_SymbolHandler.dll
            

          

          

            
❌ INVALID — missing .dll

            

              MCG_SymbolHandler\

                MCG_SymbolHandler.addin  ← .dll missing → skip
            

          

        

      

    


    
    

      
⚙️ How the Inventor BAT File Works

      
Step-by-step flow of what the bat does:


      


        

          

            
1

            

          

          

            
Check Inventor is NOT running

            
Uses tasklist to detect Inventor.exe. If running → exits with error (DLL files would be locked and fail to copy).

          

        


        

          

            
2

            

          

          

            
Define source & target paths

            
Source = folder where the bat lives (%~dp0). Target = %APPDATA%\Autodesk\Inventor 2023\Addins\. Auto-creates target if missing.

          

        


        

          

            
3

            

          

          

            
Loop every MCG_* subfolder

            
for /D iterates over directories starting with MCG_. For each, counts how many .addin and .dll files exist inside.

          

        


        

          

            
4

            

          

          

            
Validate folder content

            

              ≥1 .addin AND ≥1 .dll → proceed
              No .addin → skip + warn
              No .dll → skip + warn
              .addin count ≠ .dll count → warn
            

          

        


        

          

            
5

            

          

          

            
Clean target folder before copy

            
If target subfolder already exists → deletes all old files first. Prevents stale plugins (e.g. renamed .addin) from coexisting and causing GUID/ribbon conflicts.

          

        


        

          

            
6

            

          

          

            
Copy all files by extension

            
Sweeps the source folder for *.addin, *.dll, *.idw and copies each match to the matching subfolder in Addins. Each successful copy prints [OK].

          

        


        

          

            
7

          

          

            
Print summary

            
Reports total found, successfully activated, and any warnings. Reminds user to start Inventor 2023 to verify.

          

        


      


      

        💡 Adding a new plugin? Drop its MCG_NewPlugin\ subfolder (with at least 1 .addin + 1 .dll) into the same folder as the bat → re-run the bat. No script changes needed.
      

    


  

  
  

    
Add-in File Template (.addin)

    

      
XML Structure Overview

      
The .addin file is a manifest that Inventor uses to discover and load your plugin.

      

        

          ClassId & ClientId
          Unique GUIDs generated for your project
        

        

          Assembly
          Absolute path to C:\CustomTools\Inventor\MCG_PluginName.dll
        

        

          LoadOnStartUp
          Set to "1" for automatic loading
        

        

          UserUnloadable
          Set to "1" to allow user to unload via Add-Ins dialog
        

        

          SupportedSoftwareVersionGreaterThan
          26.. = Inventor 2023 (v27.x) and above
        

      

      

<?xml version="1.0" encoding="utf-8"?>
<!--
  File is copied automatically to:
  %APPDATA%\Autodesk\Inventor 2023\Addins\
-->
<Addin Type="Standard">

  <ClassId>{YOUR_GUID_HERE}</ClassId>
  <ClientId>{YOUR_GUID_HERE}</ClientId>

  <DisplayName>MCG_PluginName</DisplayName>
  <Description>Short description of what this plugin does</Description>

  <Assembly>MCG_PluginName.dll</Assembly>

  <!-- Inventor 2023 = version 27.x, support from 26.x -->
  <SupportedSoftwareVersionGreaterThan>26..</SupportedSoftwareVersionGreaterThan>

  <LoadOnStartUp>1</LoadOnStartUp>

  <UserUnloadable>1</UserUnloadable>

  <Hidden>0</Hidden>

</Addin>
      

      
💡 Replace MCG_PluginName in DisplayName and Assembly. The Assembly path is relative — the .addin and .dll must be in the same folder. Generate a fresh GUID for ClassId & ClientId.

Tool Overview

MacGregor Fitting Tools is an AutoCAD plugin that manages the full lifecycle of Fittings — from extracting data from Inventor 3D drawings, building a standardised Block library, designing in CAD, to exporting BOM tables and placing Balloons automatically. The plugin integrates into AutoCAD as a single PaletteSet with 4 functional tabs.

4 Functional Tabs

TAB 1 — FITTING HANDLE

BOM Export & Balloon

Scan drawing, assign Position numbers, sync to CAD, mass-balloon, export 2-sheet Excel.

TAB 2 — PROJECT CONFIG

Project Library

Browse project-specific Fittings, insert Blocks into the drawing, assign preliminary Positions.

TAB 3 — TEMPLATE (ADMIN)

Master Library & Import

Admin imports .idw from Inventor, selects BOM Type, manages the company standard library.

TAB 4 — BLOCK UTILITIES

Block Editing Tools

Rename, add/extract objects, change insertion point, bulk-replace — no Explode required.

Key Concepts

POS_NUM

An invisible Attribute embedded in every Block on insert. Links each CAD Block to its Position number in the BOM. Do not delete manually.

-tv / -fv / -sv

Block name suffixes: -tv = Top View (used for BOM counting and Balloon placement), -fv = Front View, -sv = Side View. The system only counts -tv blocks.

Panel / Detail

Panel (Structure): Large structural assembly, welded as one unit. Detail (Hull): Loose component or bought-out accessory mounted onto a Panel. Wrong classification corrupts the BOM.

Sub-BOM

Virtual accessories attached to a parent Fitting (e.g. 2 clamps + 4 bolts). When exporting BOM, accessories are indented below the parent and inherit its Position numbering.

Stacked Balloon

When a Block has a Pos string like "001,002,003" (from Sub-BOM), the system automatically generates a chain of sequential stacked balloons on the drawing.

Tab 1 — Fitting Handle: BOM & Balloon

TAB 1 — FITTING HANDLE

Scan BOM, assign Position numbers, place Balloons, export Excel

For Engineers / Drafters. Once all Fittings have been placed on the drawing (via Tab 2), use Tab 1 to complete the entire BOM workflow.

📊 Group: BOM EXPORT
Open BOM Export

Opens the BOM Matrix Export & Sync window — the central hub for the entire BOM workflow: scan the drawing, assign Position numbers, sync to CAD, and export Excel.

When to use?After all Fittings have been placed on the drawing and you need to produce the BOM for submission. This must be opened before running Scan, Assign, or Sync.
  • 1Click Open BOM Export on Tab 1
  • 2The BOM Matrix Export & Sync window appears
  • 3Select Target BOM Type: Structure (Panel) or Detail (Hull)
  • 4Proceed with Scan → Assign → Sync → Export in order
💡 The BOM window is Topmost — it always stays above AutoCAD. You can still click behind it on the drawing.

Buttons inside the BOM Matrix Export & Sync window:

Scan & Count from DrawingStep 1 — Required

Scans the entire active drawing, finds all Fitting Blocks, and displays the result as a BOM matrix (rows = Details, columns = Panels).

When to use?Every time you need to update the BOM: after adding or removing Fittings, or at the start of the BOM workflow.
  • 1Select BOM Type: Structure (Panel) or Detail (Hull)
  • 2Click Scan & Count from Drawing
  • 3The system traverses the entire Model Space and counts Blocks ending in -tv
  • 4The BOM matrix appears: rows = Detail type, columns = Panel, cells = quantity
💡 Double-count prevention: Only Blocks ending in -tv are counted. Blocks ending in -fv, -sv, or -iso are ignored entirely — each Fitting is counted exactly once regardless of how many views are on the drawing.
⚠️ If the count is lower than expected → check that -tv Blocks have been inserted. You may have only inserted -fv or -sv views.
Auto-Assign PositionsStep 2 — Required

Automatically assigns sequential Position numbers (001, 002, 003…) to all scanned Fittings. Sub-BOM accessories automatically receive the next numbers after their parent.

When to use?Immediately after a successful Scan, once you have verified the quantities are correct.
  • 1Confirm the Scan result table looks correct
  • 2Click Auto-Assign Positions
  • 3Numbers are assigned sequentially from 001
  • 4A Fitting with Sub-BOM: parent gets "001", accessories get "002", "003" — merged into string "001,002,003"
⚠️ Not yet synced to CAD! Numbers at this point only exist in the BOM window. You must click Sync Pos to CAD next.
Sync Pos to CADStep 3 — Required

Writes the Position string into the invisible POS_NUM Attribute of each Block on the drawing. After this step, Blocks on the CAD drawing "know" their Position number.

When to use?Immediately after Auto-Assign Positions, before placing any Balloons. Also run again whenever Positions change (Fittings added/removed → Scan again → Assign → Sync).
  • 1Click Sync Pos to CAD
  • 2The system iterates through every Block on the drawing, locates the hidden POS_NUM Attribute
  • 3Writes the Position string into that Attribute
  • 4Status bar shows: "Synced X blocks"
✅ After this step, Blocks on the CAD drawing carry Position numbers. Balloons can be placed at any time.
Export to ExcelStep 4 — Output

Generates the BOM Excel file automatically with 2 standard sheets: MATRIX (Panel × Detail cross-reference) and DATA (full item list).

When to use?After Scan, Assign, and Sync are complete. Use to submit the BOM or for review.
  • 1Click Export to Excel
  • 2A save dialog appears — choose folder and filename
  • 3Excel file is generated automatically with 2 sheets
Sheet MATRIX
Panel × Detail matrix. Columns = Panels, Rows = Details. Cell value = quantity of that Detail in that Panel.
Sheet DATA
Full item list. Accessories are indented and italicised. Pos column uses Text format (@) to preserve leading zeros.
🚫 NEVER edit numbers in Excel. If changes are needed → edit on CAD → Scan → Assign → Sync → Export again from scratch.
🎈 Group: BALLOONING
Place BalloonIndividual — precise placement

Place Balloons one at a time — click each -tv Block on the drawing and a Balloon appears at that exact click position.

When to use?When you need precise control over each Balloon's position — for example, a drawing with a few special Fittings that require specific annotation placement, or when only 1–2 Balloons are missing.
  • 1Ensure Sync Pos to CAD has been completed
  • 2Click Place Balloon — cursor switches to pick mode
  • 3Click on a -tv Block on the drawing
  • 4Balloon appears at the clicked position — repeat for the next Block
  • 5Press ESC to end the command
⚠️ If the Balloon appears empty or reports "POS_NUM not found" → Sync was not done. Return to BOM Export → Sync Pos to CAD.
Mass Auto-BalloonBulk — fastest option

Place Balloons in bulk by drawing a selection frame around an area. The system automatically finds all -tv Blocks inside the frame and places Balloons on them.

When to use?After all Fittings are placed and Sync Pos to CAD is done. This is the fastest way to annotate an entire drawing in a single operation.
  • 1Ensure Sync Pos to CAD has been completed
  • 2Click Mass Auto-Balloon
  • 3Draw a selection frame (click 2 corners) around the target area
  • 4System scans and finds all -tv Blocks inside the frame
  • 5Stacked Balloons appear automatically — -fv and -sv Blocks are skipped
✅ If a Block has a Pos string like "001,002,003" (from Sub-BOM), the system automatically creates 3 stacked Balloons in sequential order.
💡 You can run this command multiple times on different areas of the same drawing to control Balloon placement zone by zone.

TAB 2 — PROJECT CONFIG

Browse & insert Fittings from the project library

For Engineers / Drafters. Only displays Fittings that Admin has configured for the current project.

Open Project Library

When to use?When the Fitting list is large and you need to find something quickly by keyword.

✏️ Project Pos. Column — Preliminary Position numbering

A preliminary Position column inside the Project Library. Can be filled manually or via Auto-Assign Pos. This is a planning number, not the official drawing Position.

When to use?When you want to plan Position numbers before inserting into CAD, or to track Fittings by order in the project list.
Auto-Assign Pos

Automatically fills the Project Pos. column with sequential numbers starting from 001 for all Fittings in the list.

When to use?When you need a preliminary Position plan for the entire project before starting to design.
⚡ Main Actions
Insert to CADPrimary action

Inserts the selected Fitting's Block into the active AutoCAD drawing. The system automatically embeds the hidden POS_NUM Attribute into the Block on insertion.

When to use?Every time you need to place a Fitting on the drawing.

  • 2Click Insert to CAD — cursor transfers to the AutoCAD drawing

  • 4The Block appears at that position with the hidden POS_NUM Attribute embedded

Tab 3 — Template: Import & Master Library (Admin)

TAB 3 — TEMPLATE (ADMIN)

Import .idw from Inventor & Manage the Master Library

For Admin / Key Users only. This is where all Fittings originate — import from Inventor and build the company-standard library.

📥 Group: IMPORT TEMPLATE FROM IDW
Target BOM Type (Radio Button)Must select before Import

Select the BOM classification for the Fittings about to be imported: Panel (Structure) — large structural assembly welded as one unit; or Detail (Hull) — a loose component or bought-out accessory mounted onto a Panel.

When to use?Every time before clicking Import .idw — always set this correctly for the files you are about to import.
Panel (Structure)
Structural assemblies, welded as one. E.g. Frame Assembly, Hatch Cover Panel, Mooring Winch Base.
Detail (Hull)
Loose components, bought-out accessories, hull fittings. E.g. Cleat, Sheave, Hook, Eye Plate.
🚫 BOM Type cannot be changed after Import. If you import with the wrong type → you must delete the entry and re-import.
Import .idwAdmin primary action

Imports one or more .idw files from Inventor. The system communicates with a running Inventor instance, extracts geometry and properties, creates Blocks, and saves them to the Master Library.

When to use?When new .idw files are received from the Inventor team and need to be added to the library for Engineers to use. Typically done in batches when multiple new Fittings arrive.
  • 1Open Autodesk Inventor first (leave it idle — no file open)
  • 2Select the correct BOM Type (Panel / Detail)
  • 3Click Import .idw
  • 4Select one or more .idw files in the file dialog
  • 5The process runs in the background — TxtImportStatus updates with progress
  • 6When complete, new Fittings appear in the Master Library
💡 What the system does in the background: Reads iProperties (Part Number, Description — prefers Designer field over Author), rounds Mass value, unlocks standard layers (Mechanical-AM_3, _7, _9), crops each view to its Bounding Box, forces Linetype and Color to ByLayer, embeds hidden POS_NUM Attribute, saves to Master Library.
⚠️ Autodesk Inventor must be installed on this machine and running. If Inventor is not open → the import will time out or fail.
📖 Group: MASTER LIBRARY
Open Master Library

Opens the Master Library window — the company-wide repository of all standard Fittings. Only Admin should modify entries here.

When to use?After Import is complete to verify results; or to manage the library (add from CAD, configure Sub-BOM, add to Project, update Blocks).

Buttons inside the Master Library window

➕ Adding & Configuring Fittings
➕ Add from CADAdd manually drawn Blocks

Scans multiple views (-tv, -fv, -sv) of a Fitting drawn directly in AutoCAD and groups them under one Part ID in the Master Library.

When to use?When you have drawn a Fitting directly in AutoCAD (no .idw file exists) and want to save it to the library for reuse across projects.
  • 1Draw the Fitting Block in the drawing (with correct suffixes: -tv, -fv, -sv)
  • 2Click ➕ Add from CAD
  • 3Enter a unique Part ID
  • 4The system groups all Blocks sharing that Part ID (the different views) into one Fitting entry
  • 5The Fitting appears in the Master Library
⚠️ Part ID must be unique. Avoid AutoCAD-reserved characters: \ / : * ? " < > |. If "Duplicate name" is reported → use Rename Block (Tab 4) to rename the Block first.
🔗 Sub-BOM / AccessoriesConfigure accessories

Attach virtual accessories to a parent Fitting. Accessories have no Block on the drawing but will appear in the BOM with their own quantities and units.

When to use?When a Fitting requires accompanying accessories on installation — e.g. 1 Sheave needs 2 clamps + 4 bolts. Admin configures this so the BOM automatically calculates accessory quantities.
  • 1Select the parent Fitting in the list
  • 2Click 🔗 Sub-BOM / Accessories
  • 3The Accessory Manager window opens
  • 4Add each accessory: select from Library or create a new Virtual Item
  • 5Enter the quantity (Qty) per accessory
  • 6Save — accessories will appear in BOM with Positions following the parent
✅ In the exported BOM: parent Fitting on the main row, accessories indented below, Pos column shows the combined string "001,002,003".
🔄 Updating & Managing
Push Update

After modifying a Block directly on the drawing, use Push Update to overwrite the DWG file in the Library with the updated version.

When to use?When a Block in the Library needs correction (e.g. missing lines, wrong scale) — modify it on the CAD drawing, then Push Update to write the fix back to the Library.
  • 1Insert the Block that needs fixing onto the drawing (from Library)
  • 2Edit the Block using Tab 4 tools (Add Objects, Extract Objects, etc.)
  • 3Select the modified Block on the drawing
  • 4In the Master Library window, select the corresponding Fitting
  • 5Click Push Update
  • 6The system overwrites the DWG in the Library with the updated Block
Add to Active Project

Adds the selected Fitting(s) from the Master Library to the currently active project's list — Engineers will then see these Fittings in Tab 2.

When to use?When Admin needs to add more Fittings to an active project. The project must already be loaded in the Project Library window (Active Project shown in the top-right corner of Master Library).
  • 1Confirm the Active Project is shown correctly (top-right corner)
  • 2Select one or more Fittings in the DataGrid (Ctrl+Click for multiple)
  • 3Click Add to Active Project
  • 4Selected Fittings appear immediately in Tab 2 → Project Library
Insert to CAD

Inserts a Fitting Block from the Master Library directly into the active drawing (bypassing the Project Library).

When to use?When Admin wants to test a newly imported Fitting on the drawing to check its appearance, or needs to insert quickly without going through the Project workflow.
Refresh

Reloads the entire Master Library list from the database.

When to use?After Import .idw completes or after Push Update to see the latest results.
Remove Selected

Permanently deletes the selected Fitting(s) from the Master Library.

When to use?When a Fitting was imported incorrectly (wrong BOM Type, wrong Part ID, corrupted views) and needs to be removed so it can be re-imported correctly.
🚫 This action cannot be undone. Removing from the Master Library also removes the Fitting from all projects currently using it. Think carefully before deleting.

Tab 4 — Block Utilities

TAB 4 — BLOCK UTILITIES

Advanced Block operations — no Explode required

The hidden POS_NUM Attribute is preserved through every operation. Available to both Admin and Engineers.

🔧 Group: BLOCK UTILITIES
Rename Block

Rename or clone a Block under a new name. Automatically updates any MText content inside the Block if it matches the old Block name.

When to use?
• Before Add from CAD to Library: set the correct Part ID per naming standards.
• When a Block has the wrong suffix (e.g. wrongly named -fv instead of -tv) — fix the suffix so BOM Scan and Balloon work correctly.
• When you need to clone a Block to create a new variant without modifying the original.
  • 1Click Rename Block
  • 2Click the Block to rename on the drawing
  • 3A dialog appears — enter the new name
  • 4Choose: Rename (change name in-place) or Clone (create a copy with the new name)
  • 5All instances of the same Block name on the drawing are updated
💡 Clone creates a new Block with its own separate definition — editing the clone does not affect the original Block.
Redefine Blocks

Overwrite Block definitions in the current drawing using definitions from another open drawing. All instances in the current drawing update to the new definition.

When to use?When the Library has been updated (Push Update) and you need to update instances already on the drawing without deleting and re-inserting each one manually.
  • 1Have the source drawing (with the updated Block definition) open alongside the current drawing
  • 2Click Redefine Blocks
  • 3Select the source drawing from the list
  • 4All same-name Blocks in the current drawing are updated to the new definition
⚠️ The source drawing must be open in AutoCAD — you cannot select a closed file.
Replace BlockSmart spatial replace

Bulk-replace a source Block with a target Block using spatial position matching (Bounding Box). Preserves the original Block's Scale and Rotation.

When to use?When the entire drawing uses an old Fitting that needs to be swapped for a new version. E.g. 15 instances of Sheave-A on the drawing — replace all with Sheave-B in one step without manually deleting and re-placing each one.
  • 1Click Replace Block
  • 2Click the source Block (the one to be replaced)
  • 3Click the target Block (the replacement)
  • 4The system uses Bounding Box matching to find all instances and replace them
  • 5Original Scale and Rotation are preserved
💡 The hidden POS_NUM Attribute is carried over from the old Block to the new Block after Replace.
Change Insertion Point

Change the Base Point of a Block. The system automatically compensates coordinates for all instances of the same Block name on the drawing so they do not appear to shift.

When to use?When a Block's Base Point is in the wrong location (e.g. at a corner instead of the centre) making alignment difficult on insert. Fix it once and it applies to all instances automatically.
  • 1Click Change Insertion Point
  • 2Click the Block whose Base Point needs changing
  • 3Click the new desired Base Point (use snap for precision)
  • 4The system updates the Block definition and compensates all instances' coordinates
⚠️ All same-name Blocks are affected — this is the expected behaviour. If it looks like a shift, use Zoom to Extents to see the full picture. Press Ctrl+Z to Undo if needed.
Add Objects to Block

Merges loose objects (Lines, Arcs, Hatches…) into an existing Block without breaking the Block or losing its Attributes.

When to use?When a Block is missing some geometry — e.g. a line or circle was accidentally omitted when the Block was first created. Instead of Exploding and recreating, use Add Objects to append geometry quickly.
  • 1Draw the missing objects at the correct position on the drawing
  • 2Click Add Objects to Block
  • 3Click the target Block
  • 4Click the loose objects to merge
  • 5Press Enter to confirm — coordinates are automatically compensated
✅ Objects are merged into the Block — all same-name instances on the drawing also update.
Extract Objects from Block

Pulls objects from inside a Block out to Model Space without Exploding the Block. Block structure and Attributes remain intact.

When to use?When you need to take a line or arc out of a Block for reference, measurement, or reuse elsewhere. Also useful when a Block contains unwanted geometry that needs to be removed without Exploding.
  • 1Click Extract Objects from Block
  • 2Click the objects inside the Block that you want to extract
  • 3Press Enter to confirm
  • 4Objects appear in Model Space at their correct positions — Block remains intact
💡 The Block is not Exploded — other instances of the same Block name on the drawing are not affected.
🗂️ Group: DRAWING COLLECTION
Collect Drawings (.dwg)Merge multiple drawings into one

Merges Model Space content from multiple .dwg files into the current drawing, laid out horizontally, with Blocks auto-renamed to prevent name clashes.

When to use?When you need to consolidate multiple individual Panel drawings into one combined MTO (Material Take-Off) drawing to export a full-ship BOM.
  • 1Open the destination drawing (the file that will contain all merged content)
  • 2Click Collect Drawings (.dwg)
  • 3Select multiple .dwg source files in the file dialog
  • 4The system imports each file: Blocks renamed to [FileName]_[BlockName], special Blocks A1 and CAS_HEAD are preserved
  • 5Horizontal layout — each drawing is positioned 100mm apart anchored to its A1 frame
  • 6Unused Block definitions are purged automatically
⚠️ If any entities lie outside the A1 frame (overflow) → the system logs a warning. Clean up source drawings before collecting to avoid this.
Collect Drawings (.idw)Merge directly from Inventor

Same logic as Collect .dwg but reads Inventor .idw files directly, converting them to AutoCAD entities before merging.

When to use?When you want to merge drawings directly from Inventor without first exporting each file to .dwg individually.
  • 1Open Autodesk Inventor (leave it idle)
  • 2Click Collect Drawings (.idw)
  • 3Select multiple .idw files
  • 4The system converts and merges — same rename and purge logic as Collect .dwg
💡 Requires Inventor to be running on this machine. If Inventor is not open the process may time out.

FAQ & Troubleshooting

I clicked Import .idw but nothing happens, or it times out?
The plugin needs to communicate with Autodesk Inventor. Check: Is Inventor installed? Is Inventor currently running? Tip: Open Inventor first (leave it idle with no file open), then run Import — this prevents timeouts and speeds up the process significantly.
Scan & Count shows fewer Fittings than what I placed on the drawing?
The system only counts Blocks ending in -tv. You may have only inserted -fv or -sv views without inserting the main -tv view. Every Fitting needs at least one -tv Block on the drawing to be counted in the BOM.
Balloon placement shows "POS_NUM not found" or places an empty Balloon?
Data has not been synchronised to the drawing. After running Scan and Auto-Assign in the BOM window, you must click Sync Pos to CAD. Correct order: Scan → Auto-Assign → Sync Pos to CAD → place Balloons.
Add from CAD to Library reports a "Duplicate name" error?
The Part ID you entered already exists in the Library. Make sure the Part ID is unique and does not contain AutoCAD-reserved characters (\ / : * ? " < > |). Use Rename Block (Tab 4) to rename the Block before adding it to the Library.
Mass Auto-Balloon places Balloons on -fv Blocks instead of -tv?
Check the Block name: it must end exactly with -tv (case-sensitive). Use Rename Block (Tab 4) to correct the suffix if needed.
I edited quantities in the Excel file — now the numbers are out of sync with CAD?
Never edit numbers in Excel. The CAD drawing is the sole source of truth. All changes must be made in CAD (add/remove Blocks), then: Scan → Auto-Assign → Sync Pos to CAD → Export Excel from scratch.
Plugin does not load after installation — MCG_Fitting command not recognised?
Check in order:
  1. Does the folder %PROGRAMDATA%\Autodesk\ApplicationPlugins\MCG_FittingManagement.bundle\ exist?
  2. Does it contain a PackageContents.xml file?
  3. Does the Contents\ subfolder contain MCG_FittingManagement.dll?
  4. Re-run Install_AutoLoadCadAddin.bat with Administrator rights.
  5. Fully restart AutoCAD (do not just close the drawing).
After Change Insertion Point, same-name Blocks appear to have shifted position?
This is the expected behaviour — the system automatically compensates coordinates for ALL same-name Blocks. If it looks like a shift, use Zoom to Extents to see the full picture. If genuinely incorrect, press Ctrl+Z immediately to Undo.
After Collect Drawings, Blocks from different files are overlapping each other?
The source files are missing an A1 frame Block — the system falls back to bounding-box layout which may calculate positions incorrectly. Ensure every source drawing has a correctly placed A1 Block before collecting.
After Add Objects to Block, the merged objects appear in the wrong position?
The system calculates object positions relative to the Block's Insertion Point. If the Insertion Point is incorrect, use Change Insertion Point (Tab 4) to correct it first, then retry Add Objects to Block.

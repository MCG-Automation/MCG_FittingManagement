Tool Overview

MacGregor Fitting Tools is an AutoCAD plugin that manages the full lifecycle of Fittings — from extracting data from Inventor 3D drawings, building a standardised Block library, designing in CAD, to exporting BOM tables and placing Balloons automatically. The plugin integrates directly into AutoCAD as a PaletteSet with 4 functional tabs.

4 Functional Tabs

TAB 1 — FITTING HANDLE

BOM Export & Balloon

Scan & count Blocks, assign Position numbers, sync back to CAD, mass-place Balloons, export 2-sheet Excel.

TAB 2 — PROJECT CONFIG

Project Library

Browse the Fitting library for the active project, insert Blocks into the drawing, assign Position numbers per item.

TAB 3 — TEMPLATE (ADMIN)

Master Library & Import

Admin imports .idw files from Inventor, selects BOM Type, manages the company-standard Fitting library.

TAB 4 — BLOCK UTILITIES

Block Editing Tools

Rename, clone, add/remove objects, change insertion point, bulk replace — no Explode required, Attributes preserved.

Key Concepts

POS_NUM

An invisible Attribute embedded in every Block on Insert. Links each CAD Block to its Position number in the BOM. Do not delete manually.

-tv / -fv / -sv

Block name suffixes: -tv = Top View (used for BOM counting and Balloon placement), -fv = Front View, -sv = Side View. The system only counts -tv blocks during BOM Scan.

Panel / Detail

Panel (Structure): Large structural assembly, welded as one unit. Detail (Hull): Loose component, hull part, or bought-out accessory mounted onto a Panel. Wrong classification corrupts the BOM tree.

Sub-BOM

Virtual accessories attached to a parent Fitting (e.g. 2 clamps + 4 bolts). When exporting BOM, accessories are automatically indented below the parent and inherit its Position numbering.

Stacked Balloon

A stacked annotation balloon chain: when 1 Fitting has multiple Pos numbers (string "001,002,003" due to Sub-BOM), the system automatically creates a chain of sequential balloons on the drawing.

Installation & Activation

01

Prepare files — Copy to local drive first

Obtain the distribution package from your Admin. The package contains:

  • MCG_FittingManagement.dll — Main plugin
  • appsettings.txt — Connection settings (database, library path…)
  • Install_AutoLoadCadAddin.bat — Automated installation script
⚠️ Do not run directly from Google Drive / OneDrive. Copy all 3 files to a local folder first:
C:\CustomTools\Autocad\
02

Run the installation script (Run as Administrator)

Right-click Install_AutoLoadCadAddin.bat → Run as administrator.

The script automatically:

  1. Creates the bundle folder: %PROGRAMDATA%\Autodesk\ApplicationPlugins\MCG_FittingManagement.bundle\Contents\
  2. Copies MCG_*.dll and appsettings.txt into Contents\
  3. Generates PackageContents.xml dynamically — AutoCAD reads this on startup
✅ If the script reports SUCCESSFULLY COMPLETED, installation is done.
03

Launch AutoCAD & Activate

Restart AutoCAD 2023. The plugin loads automatically on startup. Type the following command to open the UI:

MCG_Fitting

The "Fitting Management" Palette appears on the right side with 4 tabs.

💡 Hide the Palette: Type MCG_Fitting_Hide to close without losing settings.
!

Common installation errors

SymptomCauseFix

Quick Start — 5-Step Workflow

Prerequisites: Plugin installed. Admin has imported Fittings into Master Library and configured the project. You have the AutoCAD drawing open and ready to design.
1

Open the Palette

Type MCG_Fitting in the AutoCAD Command Line. The "Fitting Management" Palette appears.

2

Insert Fittings into the drawing

Tab 2 Click Open Project Library → Find the Fitting → Insert to CAD. Repeat for all required Fittings (insert all necessary views: -tv, -fv, -sv).

3

Export BOM & assign Position numbers

Tab 1 Click Open BOM Export. In the BOM window, follow this order:

  1. Click Scan & Count — scans the entire drawing
  2. Click Auto-Assign Positions — assigns Pos numbers automatically
  3. Click Sync Pos to CAD — writes numbers into the hidden Attributes on the drawing
4

Place Balloons in bulk

Tab 1 Click Mass Auto-Balloon. Draw a selection frame around the area. Stacked Balloons appear automatically on all -tv Blocks.

5

Export Excel

In the BOM Export window, click Export to Excel. The Excel file is generated with 2 sheets: Matrix (Panel × Detail matrix) and Data (full list — accessories indented and italicised).

✅ Result: Drawing has full Balloon annotations + Excel BOM ready to submit. No manual number editing required.

BOM Export Window — Step by Step

1

Scan & Count — Scan the drawing

Click Scan & Count. The X-Ray algorithm scans through the entire drawing and counts all Fitting Blocks.

Double-Count Prevention: The system only counts Blocks whose name ends with -tv (Top View). Blocks ending in -fv, -sv, or -iso are ignored entirely — ensuring each Fitting is counted exactly once regardless of how many views are on the drawing.

If the count seems low, check that -tv Blocks have been inserted — not just -fv or -sv.

2

Auto-Assign Positions — Assign numbers automatically

Click Auto-Assign Positions. The system assigns sequential Position numbers (001, 002, 003…) to all scanned Fittings.

  • Sub-BOM accessories automatically inherit numbers after the parent
  • Parent + 2 accessories → merged string "001,002,003"
⚠️ After this step, you must click Sync Pos to CAD. Skipping this step will cause Balloons to report "POS_NUM not found".
3

Sync Pos to CAD — Write numbers to the drawing

Click Sync Pos to CAD. The system writes the Position string into the hidden POS_NUM Attribute of each Block on the drawing.

✅ After this step, Blocks on the CAD drawing carry Position numbers. Balloons can be placed at any time.
4

Export to Excel — Generate the BOM

Click Export to Excel. An Excel file is created automatically with 2 sheets:

Sheet: MATRIX
Panel × Detail matrix. Columns = Panels, Rows = Details. Cell value = quantity of that Detail used in that Panel.
Sheet: DATA
Full item list. Accessories are indented and italicised. Pos column forced to Text format (@) to preserve leading zeros.

Balloon Placement — Two Modes Compared

Place Balloon (individual)

  1. Click Place Balloon
  2. Click each -tv Block on the drawing
  3. Balloon appears at the clicked position
  4. Press ESC to end the command
Slower than Mass mode — use when precise balloon positions are needed.

Mass Auto-Balloon (bulk)

  1. Click Mass Auto-Balloon
  2. Draw a frame around the target area
  3. System scans all -tv Blocks inside the frame
  4. Stacked Balloons appear automatically
Fastest option — non--tv Blocks are skipped automatically.
Stacked Balloon: When a Block has a Pos string like "001,002,003" (from Sub-BOM), the system automatically generates 3 stacked balloons in numerical order.

Importing .idw files from Inventor

1

Select BOM Type — REQUIRED before Import

Panel (Structure)
Large structural assembly, welded as one piece. Typically contains multiple Details. E.g. Frame Assembly, Hatch Cover Panel.
Detail (Hull)
Loose component, hull part, or bought-out accessory mounted onto a Panel. E.g. Cleat, Sheave, Hook.
Wrong classification = corrupted BOM structure. BOM Type cannot be changed after Import. If imported incorrectly → delete and re-import.
2

Import .idw files

Click Import .idw files → Select one or more .idw files. The process runs in the background (async).

What the system does in the background:
  • Communicates with Autodesk Inventor to read drawing data
  • Reads iProperties: Part Number, Description — prioritises Designer field over Author
  • Rounds the Mass value to standard precision
  • Unlocks standard layers: Mechanical-AM_3, _7, _9
  • Crops each drawing view using its Bounding Box
  • Forces Linetype and Color to ByLayer
  • Embeds hidden POS_NUM Attribute into each Block
  • Saves to Master Library
💡 Open Inventor first (leave it idle with no file open) before clicking Import — reduces timeout risk and speeds up the process.

Block Utilities (Tab 4)

TAB 4 — BLOCK UTILITIES

Advanced Block operations — no Explode required

All hidden Attributes (POS_NUM) are preserved throughout every operation. Divided into 2 groups: Block Utilities and Drawing Collection.

Group 1 — Block Utilities

Rename Block

Rename or clone a Block with a new name.

  • Automatically updates any MText content inside the Block
  • Use before Add to Library to set the correct Part ID

Add Objects to Block

Add loose objects into an existing Block without breaking it.

  • Click command → Select target Block → Select objects to merge
  • Coordinates are automatically compensated — no positional drift

Extract Objects from Block

Pull objects from inside a Block out to Model Space.

  • Click command → Click the geometry lines inside the Block
  • Block is not Exploded — Attributes remain intact

Change Insertion Point

Change the Base Point of a Block.

  • Automatically compensates coordinates for all same-name Blocks on the drawing
  • No need to re-adjust each Block instance individually

Smart Replace (Spatial)

Bulk-replace Blocks based on spatial position.

  • Select source and target Blocks
  • Uses Bounding Box for spatial matching and automatic replacement
  • Preserves original Block Scale and Rotation

Redefine Blocks

Sync Block definitions from another open drawing.

  • Select the source drawing (must be open in AutoCAD)
  • Safely updates Block definitions without breaking existing instances

Group 2 — Drawing Collection

Merge content from multiple drawings into a single consolidated file. Used to combine multiple panels into one MTO drawing.

Collect Drawings (.dwg)

  • Select multiple .dwg files
  • Blocks renamed: [FileName]_[BlockName]
  • Special Blocks A1 and CAS_HEAD are preserved
  • Unused Block definitions are purged
  • Horizontal layout with 100 mm gap between frames
  • Warning issued if limits are exceeded

Collect Drawings (.idw)

  • Same logic as Collect .dwg
  • Reads Inventor .idw files directly
  • Automatically converts to AutoCAD entities
  • Same rename and purge logic applied

FAQ & Troubleshooting

I clicked Import .idw files but nothing happens, or it times out?
The plugin needs to communicate with Autodesk Inventor — make sure Inventor is installed on the machine. Tip: Open Inventor first (leave it idle with no file open) before clicking Import. This prevents timeouts and speeds up the process significantly.
After Scan & Count, the quantity is lower than what I drew on the CAD?
The system only counts Blocks ending in -tv. Check whether you have only inserted -fv or -sv views without inserting the main -tv view. Every Fitting needs at least one -tv Block on the drawing to be counted.
Balloon placement reports "POS_NUM not found" or places an empty balloon?
Data has not been synchronised. After running Scan and Auto-Assign in the BOM window, you must click Sync Pos to CAD to write the numbers into the hidden Attributes. Correct order: Scan → Auto-Assign → Sync Pos to CAD → Balloon.
When using Add from CAD to the Library, it reports a "Duplicate name" error?
The Part ID you entered already exists in the Library. Ensure the Part ID field is unique and does not contain AutoCAD-reserved characters (\ / : * ? " < > |). Use the Rename Block command (Tab 4) to rename the Block before adding it to the Library.
Balloons are placed in the wrong position, or on a -fv Block instead of -tv?
Mass Auto-Balloon only places on Blocks ending in -tv. Verify the Block name ends exactly with -tv (case-sensitive). Use Rename Block (Tab 4) to correct the name if needed.
I edited quantities in the Excel file, now the numbers are out of sync with CAD?
Do not edit numbers in Excel. The CAD drawing is the sole source of truth. All changes must be made in CAD (add/remove Blocks), then: Scan → Auto-Assign → Sync → Export Excel again from scratch.
Plugin does not load after installation — MCG_Fitting command not recognised?
Check in this order:
  1. Does the folder %PROGRAMDATA%\Autodesk\ApplicationPlugins\MCG_FittingManagement.bundle\ exist?
  2. Does it contain a PackageContents.xml file?
  3. Does the Contents\ subfolder contain MCG_FittingManagement.dll?
  4. Re-run Install_AutoLoadCadAddin.bat with Administrator rights.
  5. Fully restart AutoCAD (do not just close the drawing).
After using Change Insertion Point, same-name Blocks appear to have shifted position?
This is the expected behaviour — the system automatically compensates coordinates for ALL same-name Blocks. If it looks like a shift, use Zoom to Extents to see the full picture. If genuinely incorrect, press Ctrl+Z immediately to Undo.
